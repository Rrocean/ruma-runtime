# Human Simulation

## 目标

不是只跑单元测试，而是像真人一样真实使用一次功能面。

## 最低覆盖

- 打开主页面并确认首屏正常渲染。
- 点击每个主要交互控件：筛选、切换、按钮、链接、关闭动作。
- 同一功能至少走两种关闭或返回路径，例如 `Esc` 和遮罩点击。
- 验证复制、搜索、空筛选、错误态和移动端视口。
- 对安装型产品，实际执行安装脚本并检查目标目录产物。

## UI 任务必做项

1. 盘点所有可点击控件。
2. 为每个控件定义“点下去应该发生什么”。
3. 用 Playwright 或真实浏览器逐项触发。
4. 记录：是否可见、是否可点击、是否产生预期状态。
5. 至少检查一个窄屏视口。

## 推荐输出

```text
[Human Simulation]
覆盖范围：
- ...

已点击控件：
- ...

异常：
- ...

截图/证据：
- ...
```

## 失败时怎么做

- 按控件分组，不要只说“页面有问题”。
- 最小复现到一个页面、一个控件、一个状态切换。
- 补一条自动化用例防回归。
