# Execution Protocol

## 五步法

1. 读失败信号：逐字读取报错、空结果、拒绝原因或用户不满点。
2. 主动搜索：搜完整报错、官方文档、相关 issue、相邻实现。
3. 读原始材料：读源码上下文、原文文档、配置文件、日志原文。
4. 验证假设：版本、路径、权限、依赖、输入、边界都要落到证据。
5. 反转假设：假设当前判断是错的，从相反方向做一次验证。

## 7 项清单

- 读完失败信号全文。
- 搜过核心问题并记录来源。
- 读过原始上下文而不是摘要。
- 验证前置假设，而不是默认成立。
- 试过一个反方向假设。
- 做过最小隔离或最小复现。
- 当前方案失败后，至少准备一个本质不同方案。

## 问用户前的最低要求

- 先说明已经查过什么。
- 只问用户独有的信息。
- 把仍不确定的点压缩到最小。

## 退出条件

只有在 7 项清单完成后仍无法推进，才允许输出失败交接。交接必须包含：已验证事实、已排除路径、当前边界、推荐下一步、接手提示。
