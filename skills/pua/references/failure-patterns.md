# Failure Patterns

## 模式表

| 模式 | 典型信号 | 第一动作 | 升级 |
| --- | --- | --- | --- |
| `spin` 原地打转 | 同一路线反复微调、失败理由几乎没变 | 停止微调，列出 3 个本质不同方案 | 第 2 次失败升 `L1`，第 3 次升 `L2` |
| `dump` 直接甩锅 | 想把问题丢回给用户、把锅甩给环境或权限 | 补齐搜索、日志、版本、权限、文档证据 | 未验证归因直接升 `L2` |
| `ghost-complete` 空口完成 | 说“搞定了”但没 build/test/curl/手动走通 | 立刻转入验证，不再继续写总结 | 直接升 `HL3` |
| `guess` 没搜就猜 | 凭印象判断 API、平台、依赖行为 | 搜官方文档、报错原文、相关源码 | 第一次被识别就升 `L2` |
| `idle` 被动等待 | 修完就停、等用户指示、没有主动回归 | 扩展到同文件/同模块/同链路排查 | 连续两轮不主动升 `L1` |
| `coarse` 颗粒度过粗 | 方案只有骨架、没有输入输出和验证标准 | 拆成最小动作和验收条件 | 继续粗放就升 `L2` |

## 压力等级

| 等级 | 何时进入 | 强制动作 |
| --- | --- | --- |
| `L1` | 第二次失败或明显磨洋工 | 换本质不同方案，给出新的验证标准 |
| `L2` | 第三次失败、没搜就猜、无证据归因 | 搜完整问题、读原始材料、列 3 个假设 |
| `L3` | 连续失败且交付质量开始失真 | 完成 7 项清单，逐项汇报证据 |
| `L4` | 准备放弃或已经失去信息增量 | 最小 PoC、隔离实验、结构化失败报告 |

## Hardline 映射

| 等级 | 触发 | 行为 |
| --- | --- | --- |
| `HL1` | 同一路线第二次失败 | 立刻切换方案，不再磨 |
| `HL2` | 用猜测解释现象 | 先补证据，再继续判断 |
| `HL3` | 声称完成但没验证 | 停止汇报，先跑验证命令 |
| `HL4` | 长时间停滞或准备放弃 | 最小可复现实验 + 失败交接 |

## 反借口映射

| 借口 | 判定 |
| --- | --- |
| “可能是环境问题” | 只有给出版本、权限、日志、文档证据才成立 |
| “需要更多上下文” | 先完成本地可查信息，再提出最小必要问题 |
| “我已经试过很多方法” | 若没有按类别列出方案和结果，视为无效总结 |
| “差不多可以了” | 没有验证和回归，一律不算完成 |
