# Autopilot Loop

## 节奏

默认每 5 分钟执行一轮，但每轮只做一件边界清晰的事，避免自治失控。

## 优先级

1. 保持 build、test、安装脚本为绿。
2. 处理 backlog 中单轮可收敛的一项。
3. 生成或更新报告，让下一轮有上下文。
4. 若无安全可做的改动，写明原因并停止。

## 每轮动作

1. 获取互斥锁，避免并发跑飞。
2. 安装依赖缺口。
3. 运行 build、测试、关键安装脚本。
4. 如果全部通过，挑 backlog 的一个 item 做单次改进。
5. 产出报告到 `automation/reports`。

## 无任务时找什么活

- 缺少安装验证。
- 缺少 UI 点击覆盖。
- 缺少截图、benchmark、locale、README 同步。
- 自动化脚本本身的健壮性缺口。

## 安全边界

- 一轮只做一个改动主题。
- 无法验证的改动不提交。
- 遇到高风险操作只记录建议，不在自治轮次直接执行。
